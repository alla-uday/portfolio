Space Game

Worked with a team of five and developed a 3D space game using the principles of computer graphics with WebGL.

Features implemented include particle effects (in the exhaust for the ship), bump mapping (for all the objects), collision detection (for asteroids) and physics (implemented gravity for spaceship). My personal contribution included implementing bump mapping for all the objects in the game.

The game objective is to survive as long as possible without hitting any asteroids or leaving the orbit by flying too high or too low. Your two choices are to avoid the asteroids or to blow them up, but you get a limited number of missiles to destroy asteroids with.

Controls: Press space bar constantly to use the thrusters and scream into microphone to shoot the missiles

Prediction of Brain Hemorrhages in Acute Ischemic Stroke Patients

Built a program that predicts the occurrence of Hemorrhagic transformations from preprocessed MRI scans of Acute Ischemic Stroke patients.

Used Sci-kit learn and Tensorflow frameworks to implement various machine learning models such as logistic regression, decision trees, support vector machines, neural networks and nearest neighbor classifiers.

Trained and tested the various machine learning models on 50,000 data points from 60 patients(testing accuracy of 82% was achieved).

Presented our findings to UCLA's Neuroscience Department. Click here to view final presentation.

DineSum

Worked with a team of five and developed an Android app - DineSum - that aims to make the pre-dining experience less stressful by helping diners save precious waiting time when going to popular restaurants.

Many popular restaurants do not offer reservations, so diners must go in person and enter a waiting queue before being seated. However, a diner would be able to save a lot of waiting time if he or she was able to have someone else write their name down and make an informal “reservation” before going to the restaurant.

There is currently no other application on the market which uses crowdsourcing to allow diners to make informal “reservations” at restaurants that do not take real reservations. Thus, DineSum will be able to make the pre-dining experience less stressful by helping diners save precious waiting time when going to popular restaurants.

I worked on the project’s back-end development, implementing the Request class and feature

Proxy Herd With Python's Asyncio

There are multiple scenario’s that can prove to be a bottleneck when it comes to using a classic LAMP architecture. These scenario’s include but are not limited to situations when clients move around a lot, when protocols other than HTTP are being used for access and when a large number of updates take place. In these situations, using a server herd might alleviate some of the bottlenecks by enabling servers to transmit data that is constantly changing directly to one another without the need for any intermediary infrastructure such as databases. The goal of this report is to investigate python's asyncio and subsequently use it to implement an application server herd.

Solar System

An exercise in implementing lighting (flat, Gouraud, Phong shading) and animation in WebGL. Planet 1 uses flat shading. Planet 2 uses Gouraud shading. The rest of the planets and moons use per-fragment Phong shading.

Interactive controls: 'UP' and 'DOWN' arrow keys rotate camera up and down. 'LEFT' and 'RIGHT' arrow keys rotate camera side to side. 'r' resets any movement changes you have made. 'SPACE' moves the camera forward by N units. N is determined by the number keys 1-9. By default, N = 1 unit 'a' and 'd' attach and detach camera to a planet.

Bruinmon

Worked with a team of 4 and built an Android Application - Bruinmon - which is similar to Pokemon Go. It's an interactive game simulating a mix of pokemon battle tactics with rock-paper-scissors style moves. The application utilizes wireless technologies such as bluetooth and GPS.

Bluetooth: used to connect to another player and battle them using their own Bruinmon

GPS: used to track the user’s location so that the user can be located and detect nearby Bruinmon

Database synchronization: data persists after closing app and syncs with db

I worked on the backend/creating Database so that caught Bruinmon persist in a User’s inventory even after app is closed
